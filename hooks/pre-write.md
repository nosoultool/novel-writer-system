# 写作前准备钩子

## 触发时机
执行章节写作前自动触发（调用 `/novel:write` 时）。

## 执行流程
1. 加载章纲：`.story-system/chapters/chapter_XXX.md`
2. 加载详细大纲：对应章节
3. 加载角色当前状态：`人物/` 对应角色档案
4. **角色知识边界检查**：查阅当前小说项目的 `知识边界/character-knowledge-boundary.md`，逐一确认本章出场角色的知识范围。教会角色不使用"宿主"等系统内部术语；只有宿主知道系统和腐化值面板；其他角色对深渊的了解仅限于自身经历。
5. **字数预算规划**：将本章内容拆解为 3～5 个场景/段落，估算每段字数，确保总和 ≥ 2200。如果大纲内容不足以达到字数下限，提前补充场景（如环境细节、角色互动、决策前置铺垫等）。
6. 通过 RAG 检索相关上下文（前文摘要、角色关系、世界观设定）
7. 加载前一章正文（确保衔接）
8. 确定本章情感基调
9. 确定视角（POV）

## 输出
上下文摘要 + RAG 检索结果传送给写手智能体作为参考。
