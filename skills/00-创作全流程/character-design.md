﻿# 👤 角色设计 Skill

## 命令
`/novel:characters`

## 功能
创建和管理小说角色。

## 角色设计流程

### 1. 核心角色
- 主角（性格/动机/缺陷/成长空间）
- 主要配角（盟友/对手/恋人/导师）
- 反派（动机/魅力/威胁等级）

### 2. 角色属性模板
```
名称：
年龄：
身份/职业：
外貌特征：
性格（MBTI/特质）：
核心动机：
内心冲突：
成长弧光：
关键关系：
台词风格：
```

### 3. 成长弧光模型

角色成长不是一蹴而就的，而是分阶段变化：

| 阶段 | 说明 | 示例 |
|------|------|-------------|
| **起点** | 明确角色的缺陷/不足 | 前世尼特族，自卑又自大 |
| **觉醒** | 遇到契机，意识到需要改变 | 转世后决定认真生活 |
| **尝试** | 第一次努力，往往伴随失败 | 学魔术/剑术时的挫折 |
| **突破** | 关键事件迫使角色跨越障碍 | 转移事件后独自生存 |
| **反复** | 遇到新困难，旧习复发 | 面对强大敌人时的自我怀疑 |
| **蜕变** | 真正成长，旧我消失 | 为了保护家人挺身而出 |
| **终点** | 达成终极变化 | 成为有能力保护他人的人 |

**关键原则**：
- 每个成长阶段跨越 3-5 章，而非在一章内完成
- 成长路上要有**反复和退步**，才显得真实
- 外部事件推动内部成长（事件→冲击→反思→变化）

### 4. 角色缺陷设计

好的角色缺陷应具备：
- **缺陷**：让角色不完美（如傲慢、懦弱、偏执）
- **根源**：缺陷的来源（童年创伤、过去失败）
- **影响**：缺陷如何影响剧情走向
- **救赎路径**：如何克服（或永远无法克服）

### 5. 侧面角色独立故事线

- 主要配角应有自己的**目标、冲突和成长**
- 当他们不在主角 POV 时，他们也在过着自己的生活
- 「闲话」章节展示配角视角，增强世界真实感
- 例：配角在闲话章节中展示自己的目标和成长

### 6. 关系变化轨迹

角色关系不是固定的，而是随时间渐变：

```
陌生人 → 相识 → 合作/对立 → 信任/仇恨 → 深厚羁绊/决裂
```

每个关系节点都要有**具体事件**作为转变理由：
- 不是「他们成了好朋友」，而是「她在他最落魄时给了他一顿饭」

### 7. 角色管理
- 当前状态追踪
- 实力/能力变化
- 心理状态记录
