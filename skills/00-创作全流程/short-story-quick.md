# ⚡ 短故事快速创作 Skill

## 命令
`/novel:short`

## 功能
中短篇/超短篇故事的快速创作工作流。区别于长篇网文的逐章连载模式，短故事需要先完成全篇再投稿。

## 适用场景
- 创作番茄「千字万金」等短故事参赛作品
- 需要快速从概念到完稿的短篇创作
- 短篇投稿准备工作

---

## 一、短故事 vs 长篇网文的区别

| 维度 | 长篇网文 | 短故事 |
|------|---------|--------|
| **创作模式** | 边写边发，逐章连载 | **全篇完稿后再投稿** |
| **结构** | 无限延伸，卷-章结构 | **一次性完整结构**，起承转合闭合 |
| **节奏** | 每章要有爽点/钩子维持追读 | **全篇要有一个完整的情感曲线** |
| **角色** | 慢慢展开，长期成长 | **快速建立，变化集中** |
| **投稿** | 签约后上架 | **完稿后投责编/公共邮箱** |
| **字数** | 几十万到几百万字 | 超短8k-2.49w / 中短2.5w-8w |

## 二、创作流程

### Phase 1：概念策划（1次对话）
```
1. 确定题材和篇幅 ← 根据平台活动要求
2. 一句话梗概 ← 核心冲突+设定
3. 三个角色 ← 主角+对手+关键配角
4. 核心反转 ← 一句话说清最终真相
```

### Phase 2：大纲（1次对话）
```
1. 三幕结构分解（每幕2-3句话）
2. 章节分解（每章1-2句话）
3. 钩子布局（每章章末钩子类型）
4. 反转节点（在哪里放第1/2/3层反转）
```

### Phase 3：写作（3-5次对话）
```
1. 先写开头3章 → 确认节奏
2. 写中间主体 → 保持悬念
3. 写高潮结局 → 情感收束
4. 调整字数 → 确保符合篇幅要求
```

### Phase 4：审查（1次对话）
```
1. 一致性检查：时间线、角色、伏笔
2. 情绪曲线检查：是否符合预期
3. 字数检查：是否在目标范围内
```

### Phase 5：投稿准备（1次对话）
```
1. 生成作品简介（50-200字）
2. 打包正文（DOCX/TXT）
3. 准备投稿邮件
4. 发送至责编
```

## 三、篇幅规划指南

### 超短篇（8,000-24,900字）
```
章节数：3-10章
每章字数：1,500-2,500字
结构建议：
 - 开头1章（钩子+设定）
 - 主体2-6章（冲突升级）
 - 高潮1-2章（反转+结局）
```

### 中短篇（25,000-80,000字）
```
章节数：12-35章
每章字数：1,500-2,500字
结构建议：
 - 第一幕：3-5章（建立世界观+冲突）
 - 第二幕：6-18章（展开+升级+反转）
 - 第三幕：3-8章（高潮+结局+尾声）
```
