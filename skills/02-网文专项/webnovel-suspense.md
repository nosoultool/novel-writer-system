# 🕵️ 悬疑惊悚写作指南 Skill

## 命令
`/novel:suspense`

## 功能
悬疑惊悚题材专项写作指南。包含悬念设计、恐怖氛围营造、时间循环架构、反转技巧等。

## 适用场景
- 创作悬疑/惊悚/恐怖类短篇或长篇
- 需要设计时间循环、密室、规则怪谈等结构
- 需要制造悬念钩子和反转高潮

---

## 一、悬疑惊悚的核心机制

### 三要素公式
```
悬疑 = 信息差 + 好奇心
惊悚 = 未知威胁 + 无力感
恐怖 = 认知颠覆 + 无法逃脱
```

**信息差管理（关键）**：
```
作者知道 > 读者知道 > 角色知道
      ↓            ↓
  逐渐释放    逐步发现
```
- 保持读者知道的信息略多于主角 → 制造紧张（「他背后有人！快回头！」）
- 保持主角知道的信息略多于读者 → 制造悬念（「他为什么不说实话？」）

### 四大恐怖类型

| 类型 | 核心恐惧 | 适用题材 | 示例 |
|------|---------|---------|------|
| **心理恐怖** | 失控的自我认知 | 人格分裂、记忆操控、潜意识空间 | 「我是谁？」「这是真实吗？」 |
| **生存恐怖** | 资源/空间受限 | 密室、孤岛、废弃建筑 | 出口被封、时间有限、危险逼近 |
| **社交恐怖** | 信任崩塌 | 暴风雪山庄、团体猜疑 | 「凶手在我们之中」 |
| **超自然恐怖** | 规则之外的存在 | 鬼怪、规则怪谈、不可名状 | 无法解释的现象、无法对抗的存在 |

## 二、悬念钩子设计

### 每章钩子类型

| 类型 | 用法 | 示例 |
|------|------|------|
| **信息谜题** | 抛出无法解释的信息 | 发现一张照片，照片上有你 |
| **视觉冲击** | 突然出现的恐怖画面 | 走廊尽头有人在笑 |
| **身份冲击** | 角色身份的颠覆 | 病历上写着你的名字 |
| **规则变化** | 循环/世界的规则改变 | 钟声第二次响起 |
| **情感刀** | 角色情感的爆发 | 「我爱你——也是我害了你」 |
| **倒计时** | 时间/空间在缩小 | 安全区域在崩塌 |

### 钩子放置原则
- **开头200字** → 信息钩（异常/矛盾/冲突）
- **章中30%处** → 进展钩（新线索/新威胁）
- **章末** → 悬念钩（反转/新问题/危机升级）

### 示例（来自《死循环——午夜病房》）
```
第1章末：口袋里的纸条「第三次了」 → 信息谜题
第2章末：走廊人影在笑 → 视觉冲击
第3章末：口罩下的脸和自己一样 → 身份冲击
第8章末：地下室的门自动打开 → 恐怖悬念
第10章末：手指开始变透明 → 规则变化
```

## 三、时间循环写作法

### 循环结构设计

```
一周目：发现异常 → 死亡 → 重置
二周目：确认规则 → 收集信息 → 死亡
三周目：利用规则 → 推进调查 → 取得进展
...
终周目：揭示真相 → 面对选择 → 结束循环
```

### 四大设计原则

#### 1. 信息增量原则
每次循环必须有新信息，不能简单重复：
- 主角发现之前忽略的细节
- 不同选择导致不同分支
- 从不同角色视角理解同一事件

#### 2. 差异选择原则
```
一周目：正面探索 → 触发死亡
二周目：避开危险区域 → 发现新线索
三周目：主动制造事件 → 触发隐藏剧情
四周目：利用循环反制 → 取得关键突破
```

#### 3. 代价递增原则
每次使用循环能力都有代价，防止循环变成「无限SL」：
- **记忆损耗**：每轮忘记一些无关信息
- **空间崩塌**：可探索区域逐渐缩小
- **身体代价**：伤痛在重置后不会完全消失
- **能力疲劳**：多次使用后效果递减

#### 4. 信息差管理
```
主角知道 ⊂ 读者知道 ⊂ 作者知道
     ↓              ↓
     └── 主角揭示 → ──┘
```
- 通过「了解真相的配角」调节信息释放节奏
- 配角可以在不同循环中给出不同信息碎片
- 最终将所有碎片拼合为完整真相

### 经典循环触发方式
| 触发方式 | 说明 | 效果 |
|---------|------|------|
| 死亡触发 | 主角/全员死亡时重置 | 紧张感强，容错率低 |
| 时间触发 | 固定时限到达时重置 | 倒计时压迫感 |
| 条件触发 | 满足特定条件时重置 | 可控性强 |
| 主动触发 | 主角可自主决定重置 | 策略性高 |

## 四、反转设计

### 三层反转结构（推荐）

```
表层反转（25%处）：
→ 读者以为的「真相」被推翻
→ 示例：「他不是医生，他是患者」

中层反转（50-60%处）：
→ 更换「问题」的类型
→ 示例：「这不是案件调查，这是审判」

底层反转（80%处）：
→ 重构故事的前提假设
→ 示例：「这不是真实空间，这是潜意识」
```

### 角色反转模板

| 反转类型 | 读者以为 | 实际是 |
|---------|---------|-------|
| **身份反转** | 英雄/受害者 | 当事人/加害者 |
| **动机反转** | 邪恶/冷漠 | 保护/赎罪 |
| **关系反转** | 陌生人 | 过去有关联 |
| **能力反转** | 无害的能力 | 致命的副作用 |
| **记忆反转** | 忘记/丢失 | 选择性遗忘 |

## 五、番茄平台适配要点

### 节奏控制
- **超短篇（8k-2.49w字）**：3-8章，每章一个完整悬念循环
- **中短篇（2.5w-8w字）**：12-25章，每5章一个小高潮
- **每章1500-2500字**，段落不超过5行，对话推进为主

### 完读率优化
- 每章前200字必须有悬念钩子
- 章末必须留「不可跳过」的悬念（读者必须看下一章）
- 避免大段描述性文字，用动作和对话推进
- 信息密度高：每300字至少给一个新信息

### 推荐题材方向
| 题材 | 适合篇幅 | 开篇钩子 |
|------|---------|---------|
| 时间循环 | 中短篇 | 死亡→重置→发现不是第一次 |
| 规则怪谈 | 超短/中短 | 发现规则→打破规则→代价 |
| 暴风雪山庄 | 中短篇 | 被困→死亡→凶手在内部 |
| 记忆操控 | 中短篇 | 发现记忆对不上→我是谁？ |
| 都市怪谈 | 超短篇 | 遇到异常→调查→陷入 |
