# 💥 剧情引爆 Skill

## 命令
`/novel:booming`

## 功能
当剧情平淡/卡文时，提供高强度的剧情引爆方案。

## 适用场景
- 剧情进入平缓期，读者可能流失
- 卡文，不知道下一步怎么写
- 需要制造高潮或转折

## 引爆方案库
1. **身份揭露**：隐藏身份突然曝光
2. **背叛反转**：信任的人突然背叛
3. **危机降临**：不可抗力突然出现
4. **秘密浮现**：尘封的秘密被揭开
5. **关系突变**：情感关系骤变
6. **实力飞跃**：主角获得新力量/能力
7. **幕后黑手**：揭示一直以来的操纵者
8. **代价显现**：过去的决定带来后果
9. **联盟瓦解**：稳固的联盟突然破裂
10. **终极抉择**：两难困境迫使角色选择
11. **时间循环触发**：主角陷入时间循环，每次轮回获得新信息
12. **能力反转**：原本无害的能力显露出致命副作用
13. **信息差曝光**：读者/主角发现一直以来的认知是错的

## 时间循环/多周目技法

### 适用场景
- 悬疑解谜类需要反复调查同一事件
- 主角需要多次尝试才能突破困境
- 通过信息增量推进剧情

### 循环设计原则

#### 1. 信息增量原则
每次循环不能是简单重复，必须有新信息加入：
- 主角发现之前忽略的细节
- 不同选择导致不同的结果
- 从不同角色视角理解同一事件

#### 2. 循环差异原则
每次循环必须有明确的不同：
```
一周目：发现异常 → 死亡
二周目：意识到危机 → 收集线索 → 死亡
三周目：开始反击 → 找到盟友 → 取得进展
四周目：全面对抗 → 揭示真相 → …
```

#### 3. 代价递增原则
每次使用循环能力都有代价：
- 记忆损耗（忘记上一轮的关键信息）
- 身体代价（每轮循环消耗生命力）
- 蝴蝶效应（改变过去的不可预知后果）
- 能力疲劳（多次使用后效果递减）

#### 4. 信息差管理
```
主角知道 ⊂ 读者知道 ⊂ 作者知道
     ↓              ↓
     └── 主角揭示 → ──┘
```
- 保持读者和主角之间的信息差，制造悬念
- 反派知道的信息量是一个关键变量
- 通过"了解真相的配角"调节信息释放节奏

### 循环触发设计
- **主动触发**：主角可以自主决定是否重来
- **死亡触发**：主角死亡自动回档
- **条件触发**：满足特定条件时触发
- **有限次数**：总共只能使用 N 次
1. 提供当前剧情概况
2. 系统推荐 3 个引爆方向（含示例）
3. 用户选择或组合
4. 整合到正文
