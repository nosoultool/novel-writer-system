# 🔬 设定质检 Skill

## 命令

`/novel:qa`

## 功能

在动笔之前，对世界观、角色设定、大纲进行全面的交叉验证和逻辑压力测试。充当"写作前的最后一道质检门"。

## 前置条件

- 小说项目已完成世界观搭建、角色设计、大纲规划
- 以下目录已存在：`设定集/`、`人物/`、`大纲/`

## 工作流程

1. **加载数据**：读取当前小说项目的三个支柱目录
   - `设定集/` → 世界观设定
   - `人物/` → 角色档案
   - `大纲/` → 章节规划和伏笔登记
2. **加载知识库**：读取 `agents/knowledge/setting-qa/` 下的5个检查标准文件
3. **启动质检 Agent**：调用设定质检员智能体，传入全部上下文
4. **对话式验证**：质检员与用户进行苏格拉底式对话，对存疑设定逐项探讨
5. **输出报告**：生成 `审查报告/setting-qa-report.md`

## 输出

- 结构化 QA 报告，包含通过项、关注项、问题项、优化建议
- 与用户的对话记录，展示质疑过程和优化方向

## 集成方式

### 手动调用

在完成 `/novel:outline` 之后、`/novel:write` 之前，主动调用：

```
输入：/novel:qa
```

### 推荐流程

```
设定完成 → /novel:qa ← 在此验证 → 确认无误 → /novel:write
```
