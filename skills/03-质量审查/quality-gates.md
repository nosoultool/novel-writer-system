# 🚧 质量门禁 Skill

## 命令
`/novel:quality`

## 功能
6 道质量关卡，确保章节通过方可标记完成。

## 6 道关卡

### 第 1 关：基础规范（必须）
- [ ] 无错别字、语病
- [ ] 标点符号使用正确（全角中文标点）
- [ ] 章节字数在 2000-4000 纯中文字符之间
- [ ] 段落长度适中（不过长也不过碎）

### 第 2 关：叙事结构（必须）
- [ ] 有明确的开端、发展、结尾
- [ ] 开篇 200 字内有钩子
- [ ] 结尾有留白/悬念，非总结式收尾
- [ ] 每章至少有一个情感起伏
- [ ] 视角清晰一致，无混乱切换

### 第 3 关：角色逻辑（必须）
- [ ] 角色行为符合人设
- [ ] 对话有角色区分度（不看标签也能识别）
- [ ] 角色情感变化有充分铺垫
- [ ] 角色行事动机合理

### 第 4 关：情节质量（必须）
- [ ] 本章对主线/支线有推进作用
- [ ] 有足够的冲突或张力
- [ ] 信息量适中（不灌水也不赶进度）
- [ ] 结尾有继续阅读的动力（悬念/期待/疑问）

### 第 5 关：语言质感（深度检测）
- [ ] 无 AI 味高频词泛滥（然而/但是/因此/不过/突然/似乎）
- [ ] 情感通过动作或环境展示，而非直接告知（Show not Tell）
- [ ] 对话与动作/神态自然穿插，避免纯对话墙
- [ ] 设定信息通过行动/对话展示，而非直接说明
- [ ] 叙事比例合理（叙述:独白:对话:描写 ≈ 4:3:2:1）

### 第 6 关：一致性（交叉检查）
- [ ] 与已写章节无时间线矛盾
- [ ] 与世界观设定无冲突
- [ ] 与角色当前状态一致
- [ ] 伏笔使用/回收符合原计划
- [ ] 与前文衔接自然（无跳跃感）
