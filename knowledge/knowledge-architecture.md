# 灵境知识库架构

## 分层原则

```
灵境系统（通用知识）
  ├── 标点用法、写作技巧、时代背景等
  └── 所有小说项目共享，不随具体小说变化

小说项目（专项知识）
  ├── 角色知识边界、人物档案、大纲、设定集
  ├── 章节正文、摘要、审查报告
  └── 仅当前小说使用，不同小说各自独立
```

## 通用知识库（灵境系统内）

路径：`knowledge/`

| 文件 | 内容 | 适用范围 |
|:----|:----|:---------|
| `punctuation-guide.md` | 标点符号用法规范 | 所有小说 |
| `writing-craft.md` | 基础写作技巧 | 所有小说 |
| `writing-craft-enhanced.md` | 进阶写作技巧 | 所有小说 |
| `knowledge-architecture.md` | 本文件——架构说明 | 系统自身 |
| `genre/` | 各类型小说写作知识 | 按需加载 |

## 专项知识库（小说项目内）

路径：`小说项目/知识边界/`

| 文件 | 内容 |
|:----|:----|
| `character-knowledge-boundary.md` | 角色知识边界地图 |

其他专项文件直接放在小说项目对应目录：
- `人物/` — 角色档案
- `大纲/` — 总纲、卷纲、伏笔追踪
- `设定集/` — 世界观、力量体系
- `章节/` — 正文

## 写作时查询规则

```
写前准备时：
  1. 通用写法规范 → knowledge/punctuation-guide.md（灵境系统内）
  2. 通用写作技巧 → knowledge/writing-craft.md（灵境系统内）
  3. 小说专有设定 → 小说项目/大纲/、小说项目/设定集/（小说项目内）
  4. 角色知识边界 → 小说项目/知识边界/character-knowledge-boundary.md（小说项目内）

产生新知识时：
  1. 判断是通用还是专项 → 参考 knowledge/knowledge-classification.md
  2. 自动放入对应目录，无需询问
```
