# 🏛️ 时代一致性审查 Skill

## 命令
`/novel:era`

## 功能
检查章节内容中的科技/知识/制度要素是否符合世界设定和时代背景。

## 前置准备
1. 加载 `.era-knowledge/` 下的三个知识库文件
2. 加载待审查章节正文

## 审查清单

### 1. 物质层面
- [ ] 武器/装备是否超纲？（火药、精密机械）
- [ ] 建筑/工程是否超纲？（混凝土一次成功？）
- [ ] 交通工具是否合理？（没有汽车/火车）
- [ ] 日常用品是否符合时代？（塑料/橡胶不存在）

### 2. 制度层面
- [ ] 管理制度是否有穿越感太强？（OKR/KPI 属于出戏）
- [ ] 军事组织是否合理？（三三制/现代军衔出现在中世纪？）
- [ ] 商业模式是否跨时代？（股份制/期货在异世界？）

### 3. 概念层面
- [ ] 主角的用词是否太现代？（"这个bug需要修一下"可接受，"我们来开个sprint回顾会议"不可）
- [ ] 科学知识是否准确但不越界？（"水烧开了喝不拉肚子"可，"大肠杆菌是革兰氏阴性菌"不可）
- [ ] 数学应用是否合理？（加减乘除可，微积分/线性代数不可）

### 4. 语言层面
- [ ] 角色的对话是否使用了超越他们认知的词汇？
- [ ] 叙述者的语言（第三人称）是否包含了世界不存在的概念？

## 输出格式

审查完成后输出标注报告，格式如下：

```
## 时代一致性标注：第XX章

### 🔴 明显违和
- [位置] 要素描述 → 建议方案

### 🟡 需合理化
- [位置] 要素描述 → 合理化建议

### 🟢 合理可用
- [位置] 要素描述 → 已通过
```

## 集成方式
- 自动集成在 `hooks/pre-review.md` 触发
- 也可在写作前主动调用 `/novel:era` 预检
