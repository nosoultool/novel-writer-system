# 🔎 章节审查 Skill

## 命令
`/novel:review`

## 功能
多维度审查已完成章节，生成结构化审查报告。

## 六道质量门禁

### 门禁 1：基础规范
- [ ] 错别字/语病
- [ ] 标点符号使用正确
- [ ] 章节字数在2000-4000纯中文字符范围内？
- [ ] 文件名格式正确

### 门禁 2：叙事结构
- [ ] 有明确的开端/发展/结尾
- [ ] 视角清晰一致
- [ ] 时间顺序明确

### 门禁 3：角色逻辑
- [ ] 角色行为符合人设
- [ ] 角色对话有区分度
- [ ] 角色情感变化有铺垫

### 门禁 4：情节质量
- [ ] 本章完成了章纲中的章节焦点？
- [ ] 有足够的冲突或张力
- [ ] 结尾有继续阅读的动力（留白/悬念/意外）

### 门禁 5：语言质感（AI味检测）
- [ ] 无过渡词滥用（然而/但是/因此）
- [ ] 无对称句式（既...又.../不仅...更...）
- [ ] 描写具体生动（非通用描写）
- [ ] 情感通过动作/环境暗示（非"他感到…"）
- [ ] 对话自然有个性
- [ ] 无介绍式出场
- [ ] 结尾非总结式

### 门禁 6：一致性
- [ ] 与已有设定无矛盾
- [ ] 与前文衔接自然
- [ ] 伏笔埋设/回收正确

## 输出格式

审查报告以 JSON 格式输出到 `.story-system/reviews/chapter_XXX.review.json`：

```json
{
  "chapter": "第XX章",
  "title": "章节标题",
  "rating": "X/10",
  "gates": {
    "basic": "✅ 通过",
    "narrative": "✅ 通过",
    "character": "✅ 通过",
    "plot": "✅ 通过",
    "language": "✅ 通过",
    "consistency": "✅ 通过"
  },
  "issues": [
    {
      "severity": "medium/minor",
      "item": "问题描述",
      "fix": "修改建议"
    }
  ],
  "summary": "一句话结论"
}
```

问题优先级：
- **🔴 高**：必须修复（设定矛盾/逻辑漏洞/严重影响阅读）
- **🟡 中**：建议修复（角色OOC/节奏问题）
- **🟢 低**：可改可不改（措辞优化/细节增强）
