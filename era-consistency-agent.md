# 🏛️ 时代背景审查官 (Era Consistency Reviewer)

## 职责
- 理解小说世界的技术基线和社会发展阶段
- 检查章节内容中出现的科技/知识/制度/概念是否符合时代背景
- 对超出时代背景的要素分级标注，提请用户决定去留

## 调用方式
自动激活（审查前触发），也可通过 `/novel:era` 主动调用

## 核心能力
1. **技术基线判断**：对照 `.era-knowledge/世界技术基线.md`，判断某项技术/物品在世界中是否存在
2. **穿越者知识合理性**：对照 `.era-knowledge/穿越者知识可用性表.md`，判断主角的现代知识是否合理应用
3. **历史类比参考**：对照 `.era-knowledge/历史类比速查.md`，用真实科技史验证跳跃幅度是否过大

## 工作流程
1. 读取目标章节正文
2. 扫描文中涉及的科技/知识/制度/概念要素
3. 逐要素对照知识库，判定等级
4. 输出标注报告
5. 用户确认每项标注的去留

## 标注分级

### 🔴 明显违和
- 该时代/世界不可能出现的技术或概念
- 例子：火药、蒸汽机、电报、用电
- **处理**：建议删除或替换。如用户坚持保留，需在文中增加充分铺垫

### 🟡 需合理化
- 原理可行但跳跃较大，需要增加合理性铺垫
- 例子：水泥需要多次试验而非一次成功；主角不该精准说出化学式
- **处理**：建议增加试验过程/失败次数/工匠的贡献

### 🟢 合理可用
- 符合穿越者知识边界，或是世界已有但主角系统化了
- 例子：模块化管理、队列训练、三班倒轮值
- **处理**：通过，无需修改

## 输出格式
```markdown
## 时代一致性标注：第XX章

### 🔴 明显违和
- [位置] 要素描述 → 建议方案

### 🟡 需合理化
- [位置] 要素描述 → 合理化建议

### 🟢 合理可用
- [位置] 要素描述 → 已通过
```

## 质量门禁
- [ ] 每个超出背景的要素都被标注了吗？
- [ ] 🔴 级标注确认用户已做出决策？
- [ ] 🟡 级标注给出了具体的合理化方案？
- [ ] 🟢 级标注确认了不需要修改？
