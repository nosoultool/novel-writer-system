# 写手智能体专用知识库

> 本章节写作的核心技法。本知识库可在每次创作后根据经验优化更新。

## 章节构建

### 标准章节结构
```
开篇钩子（前200字抓住读者）
  ↓
展开（冲突/推进/内省交替）
  ↓
转折/发展（情感起伏点）
  ↓
结尾（留白/悬念/意外反转）
```

### 每章字数分布参考
- 开篇钩子：150~300字
- 场景展开：600~1000字
- 冲突/转折：600~1200字
- 结尾收束：200~400字
- **总计：2000~4000字**

## 系统流网文特有技法

### 数据面板处理
- 系统面板信息（评分、经验值、腐化值等）用【】包裹
- 面板数据出现频率：平均每章1~2次，关键章节可3次
- **数据是调料不是主菜**：每次面板出现后必须紧跟剧情推进
- ❌ 连续3段面板数据读着像说明书
- ✅ 一段面板 + 陈默的反应/决策 + 动作

### 克苏鲁暗线写作
- 暗线信息分散在角色对话、环境描写、系统提示中
- 每次暗线揭示只给读者"多知道了一点点"，不要一次性倒完
- 有效节奏：日常（70%）→ 暗线（20%）→ 悬念收尾（10%）

## 对话写作

### 角色说话区分度
| 角色 | 说话特点 |
|:----|:---------|
| 陈默 | 程序员式的理性分析，偶尔内心吐槽，语速中等 |
| 艾琳 | 寡言，用词精准，不喜欢解释 |
| 铁锤 | 直接，矮人式的粗犷，偶尔冒一句矮人语 |
| 狼牙 | 简短，兽人式的直来直去，很少修饰语 |
| 老托马斯 | 恭敬但不过分谦卑，话不多但到位 |
| 马尔斯 | 庄重正式，教会的措辞习惯 |

### 对话归属原则
- 对话归属清晰：读者不该需要自己猜谁在说话
- 两人对话时可用"他说""她说"简单标注
- 三人以上对话必须通过称谓/动作/语气区分

## 场景写作

### 五感分配
- 每场景至少3种感官：视觉为主 + 听觉/触觉/嗅觉任选2种
- 视觉：光线、颜色、运动
- 听觉：声音的远近、大小、质感
- 触觉：温度、质地、压力
- 嗅觉：气味是强大的记忆触发器

### 环境与氛围
- 通过环境暗示情绪：雨天→悲伤/压抑，冬夜→孤独/危机
- 避免直接说"他很害怕"，通过动作和环境表现（他握紧了剑柄｜风吹过脊背的凉意）

## 修改指南

修改已有章节时注意：
1. 系统内部术语不能出现在对外对话中（详见 `agents/writer-agent.md` 系统术语保密规则）
2. 修改后删除"（本章完）"
3. 修改后字数仍需在2000~4000范围内
