# ✍️ 章节写作 Skill

## 命令
`/novel:write`

## 功能
根据大纲和设定写作正文章节。

## 写作前准备
1. 加载章纲：`.story-system/chapters/chapter_XXX.md`
2. 加载详细大纲：`大纲/第N卷《卷名》详细大纲.md` 对应章节
3. 加载角色当前状态：`人物/` 对应角色档案
4. 加载前一章正文（确保衔接）
5. 确定本章情感基调
6. 确定视角（POV）

## 章节结构
```
开篇（钩子）→ 展开（冲突/推进）→ 转折/发展 → 结尾（悬念/留白）
```

## 字数规范
- **2000～4000 字**（仅计中文字符，不含标点符号、空格、数字、英文）
- 验证方式（Python）：
  ```python
  import re
  cn = re.findall(r'[\u4e00-\u9fff]', open('文件.md','r',encoding='utf-8').read())
  print(len(cn))
  ```

## 写作规范
- 开篇要有钩子（悬念/冲突/新信息）
- 每章至少一个情感起伏
- 结尾留有期待感或意外留白
- 对话符合角色性格（参考人物档案中的台词风格）
- 描写调动五感（视/听/触/嗅）
- **关闭思考模式**：输出纯净叙事文本，不含推理过程

## 写作后检查
- [ ] 字数在2000-4000纯中文字符范围内？
- [ ] 本章完成了章纲中指定的章节焦点？
- [ ] 角色行为是否符合人设？
- [ ] 结尾不是总结式收尾（应有留白/悬念/意外）？
- [ ] 是否有明显的逻辑漏洞？
- [ ] 语言是否自然流畅，无明显AI味？

## 文件命名
```
第XX章-短标题.md
```
存放路径：`章节/`
